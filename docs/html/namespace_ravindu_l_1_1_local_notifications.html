◆ LocalNotificationCollisionBehaviour

enum RavinduL.LocalNotifications.LocalNotificationCollisionBehaviour
strong

Specifies the way in which a collision between two local notifications (i.e. attempting to show one which the other still persists) is handled.

Enumerator
Wait 

The new notification gets delayed until the expiration of the current notification, following which it will be shown.

Replace 

The new notification will trigger the current notification to be hidden, following which it will be shown.

Specifies the state of a LocalNotificationPresenter.

Enumerator
Hidden 

The LocalNotificationPresenter being hidden from view, and the user not being able to interact with it.

Hiding 

The LocalNotificationPresenter transitioning to the LocalNotificationState.Hidden state from being shown.

Showing 

The LocalNotificationPresenter transitioning to the LocalNotificationState.Shown state from being hidden.

Shown 

The LocalNotificationPresenter being displayed, awaiting user interaction or to be hidden.

Restoring 

The LocalNotificationPresenter transitioning to the LocalNotificationState.Shown state, but not from being hidden.
